## PART 1: CRITICAL BUGS TO FIX IMMEDIATELY

### 1.1 Layout / Navigation Chaos
Two conflicting nav systems exist: a left sidebar (Home / Explore / Notifications / Books / Series / Challenge / Progress / Wishlist / Diary / Profile / Users / Reports / Recommend) AND an old top nav bar (LibraryMS | Dashboard | Books | Series | Challenge | Progress | Suggest | Feed). Eliminate the old top nav bar entirely. The left sidebar is the ONE persistent nav. Challenge, Wishlist, Diary, Reading Progress, and all other authenticated pages must use the unified sidebar layout. "Home" currently opens the social feed — this is fine but add a redirect so `/` for authenticated users hits `/feed`.

### 1.2 Social Feed (/feed) — Stuck on Loading
The "Following" tab shows a loading spinner indefinitely. Wrap all feed fetch calls in `try/catch` with fallback empty states:
- Following empty: "Follow some readers to see their posts here" + `[Explore Readers]` button
- Trending empty: "No trending posts yet" + `[Be the first to post]` button
- Discover empty: Show 4–6 suggested users with `[Follow]` buttons (pull from `/api/users/suggested`)
- Right panel "Trending Books" — fix the `/api/books/trending` endpoint
- Right panel "Who to follow" — fix `/api/users/suggested`

### 1.3 Explore Page (/explore) — Completely Empty
Currently shows only "Search Books & People" and a Ctrl+K hint. Replace with a rich discovery hub (see Part 3, Page Group 4).

### 1.4 Profile Page — "timep" Text Overflow
"AVG RATING" stat card shows "timep" (truncated "timepass") overflowing its container. The Reading Stats grid cells must have overflow handling. "Timepass" is the rating label — render it in amber color with proper truncation and tooltip on hover showing full value.

### 1.5 Dashboard Rainbow Progress Bars
Dashboard shows color-coded horizontal progress bars for Total Books / Users / Active Issues / Fines — these have no visual hierarchy, look broken in dark mode. Replace entirely with a clean stat card grid.

### 1.6 All Sidebar Links Non-Functional
Clicking Home, Explore, Notifications, Books, Series, Challenge, Progress, Wishlist, Diary, Profile, Users, Reports, Recommend in the sidebar either does nothing or shows a 404. Implement or wire up all routes listed in the sidebar. Every link must resolve to a functional page.

### 1.7 ML Recommendations Not Used
The `Recommendation Systems/` folder contains a trained ML model and the full Goodreads dataset but the recommendations page does not use it. Integrate the model. Load the dataset at startup. Serve personalized recommendations via `/api/recommendations/<user_id>`. Add a fallback (trending/popular) for cold-start users.

### 1.8 Charts Rendering Empty
Dashboard Monthly Trends chart, Challenge Monthly Progress chart, Reports charts — all have Chart.js canvas elements but render blank. Initialize all Chart.js instances properly in `DOMContentLoaded` blocks with real data from API calls. Never leave a canvas uninitialized.

### 1.9 Notifications Stats — Raw Text
"0 Total / 0 Unread / 0 Read" is displayed as plain text lines with no card structure. Fix with stat card grid.

### 1.10 Reports Page — One Giant Card Per Metric
Each single metric gets its own full-width card. Group into semantic sections with 2–4 metrics per row.

### 1.11 Favorite Books Grid — Broken Empty State
Shows "Search to add / Drag to reorder (you)" as plain text. Fix with proper slot UI using dashed border + plus icon.

### 1.12 Books Page — Single Column Waste
Desktop shows one book card per row. Fix: CSS Grid with `repeat(auto-fill, minmax(180px, 1fr))`.

### 1.13 Landing Page — Books Counter Shows 0
"0 BOOKS" in the live stats bar despite 11,127 books in the Goodreads dataset. Seed the stats from the Goodreads dataset on first load. `/api/seed/stats` must return correct counts.

### 1.14 "BOOKSOCIAL" Watermark Overflow
Fix: `opacity: 0.08`, `overflow: hidden`, `user-select: none`, `pointer-events: none`.

### 2.2 Semantic HTML5

Every page must use:

```
<header>                    — top app bar only
<nav role="navigation"
     aria-label="Main navigation">  — left sidebar
<main role="main">          — exactly one per view
<article>                   — each post, review, diary entry
<section aria-labelledby>   — named content regions
<figure> + <figcaption>     — all book covers
<aside>                     — right panel widgets
<footer>                    — page footer
<table> + <caption>
  + <thead> + <th scope>    — data tables
<dl><dt><dd>                — label/value pairs
```

ARIA requirements (non-negotiable):
- `aria-live="polite"` on: notification bell count, feed container, typing indicators, live search results
- `aria-pressed` on: theme toggle, grid/list toggle, follow button, like button
- `aria-label` on: all icon-only buttons
- `aria-current="page"` on active sidebar link
- `role="status"` on loading skeletons
- `aria-expanded` on collapsible sections
- `aria-selected` on active tabs
- `aria-required="true"` + `*` on required form fields
- `role="alert" aria-live="assertive"` on form error messages
- `role="tablist"` + `role="tab"` on tab components

**Backend Integration**
```python
# recommendations_service.py
import pickle, pandas as pd
from pathlib import Path

REC_MODEL_PATH = Path("Recommendation Systems/model.pkl")
BOOKS_DATA_PATH = Path("Recommendation Systems/books.csv")

# Load at app startup — not per request
with open(REC_MODEL_PATH, 'rb') as f:
    rec_model = pickle.load(f)
books_df = pd.read_csv(BOOKS_DATA_PATH)

@app.route('/api/recommendations/<user_id>')
def get_recommendations(user_id):
    reading_history = get_user_reading_history(user_id)
    if not reading_history:
        # Cold start: return popular from Goodreads dataset
        popular = books_df.nlargest(12, 'average_rating')[
            ['book_id','title','authors','average_rating','image_url']
        ].to_dict('records')
        return jsonify({'strategy': 'popular', 'books': popular})
    # Personalized
    recs = rec_model.recommend(user_id, reading_history, n=12)
    return jsonify({'strategy': 'personalized', 'books': recs})
```

## DESIGN PHILOSOPHY

BookTale must feel like a **premium reading companion**, not a generic SaaS tool.

**Aesthetic direction**: warmth (generous whitespace, soft purple gradients, sepia mode paper feel), delight (book cover parallax tilt, smooth 200ms transitions, confetti on challenge completion), intelligence (AI badges with explanation tooltips, smart empty states that suggest actions), community (social proof — "3 of your followers read this"), trust (clean data tables, honest stats).

**Color language**:
- Purple/Indigo `#6366f1` — primary brand, CTAs, active states
- Violet `#a855f7` — accent, gradients
- Pink `#ec4899` — social, likes, hearts
- Green `#22c55e` — available, on-track, success
- Amber `#eab308` — warnings, "Timepass" rating, streaks
- Red `#ef4444` — overdue, danger, blocked
- Gray — neutral elements, secondary text, borders

**Typography**:
- Headings: Inter or system-ui, weight 600–700, tight tracking
- Body: Inter or system-ui, weight 400, line-height 1.6
- Code/IDs: JetBrains Mono or Fira Code
- Sepia body: Georgia or serif
